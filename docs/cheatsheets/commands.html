Phrase Mining

phrase_miner
Iterative k-mer mining (k=25→3) with masking & de Bruijn extension.
-i input -o dir -K k_max -k k_min -n freq -t tinyids -D deBruijn -S subsumption --dedup
phrase
Original n-gram detection (simpler, single-pass).
-i -o --fields --min-words --min-ids --lemmatize --verbatim
phrase_builder
K-mer frequency visualization charts.
-i -m model -o
phrase_grouper
Bottom-up family analysis (prefix/suffix/infix).
-i -o -K -k --min-family-size --assignment

Pattern Utilities: Analysis

Multi-mode tool — one mode per invocation.

Coalesce Variants

-c FILE -o OUT — subsumption + prefix trie
--coalesce-report FILE   --min-prefix-tinyids N

Merge Patterns

-M FILE FILE -o OUT — deduplicate/merge TSVs
--merge-pattern-column   --merge-tinyids-column

Field Analysis

-A FILE -i JSON -m MODEL -o OUT
--min-field-count N   --min-tokens N   --exclude-patterns

Validate Subsumption

-V FILE -i JSON -o OUT   -w workers

Detect Rare Words

-R -i JSON -o OUT — wordfreq Zipf scores
--zipf-threshold   --caps-penalty   --no-whitelist

Pattern Utilities (cont.)

Temporal Seeds

-T -o OUT — expand 25 seeds → ~2,100 variants

Expand Verbatim

-e FILE -o OUT — generate pattern variants

TSV Editor

--edit FILE   --port N — browser-based editor

Supplementary Management

--harvest-to-supplementary FILE — auto-ingest
--promote-supplementary — local → global
--clean-local — remove promoted entries

Reporting, Data, Workflow & Options

Reporting & Analysis

discovery_report
Markdown summary of a pipeline execution.
-d dir -p {instrument,phrase} -o output --version -i json
pipeline_report
Comprehensive multi-phase execution report.
-s state_file / -d dir -o -p {1,2,3,4} --title --version
strip_report
Quality report for stripped outputs (remnant detection).
-d dir -o -i json --version --no-temporal-scan
recall_analyze
Detection recall vs ground-truth patterns.
-i -m -o -F pattern_file --pipeline-output -r report --suggest-patterns

Data Extraction

subset
Extract CDE records by tinyId or text content.
-i -o -m -l id_list / -L id_file -t text --regex -x exclude
extract_embed
Extract fields for transformer embeddings.
-i -m -o --path-file --output-format {json,csv,tsv} --concatenate
count
Count field occurrences and unique values.
-i --fields -o --match-type {non_null,null,fixed,regex} --group-by
lemma_fasta
Lemmatized text → FASTA format for sequence tools.

Workflow Orchestration

Run a Pipeline

cde-analyzer workflow run FILE.yaml \
  -s input_json=data.json \
  -s output_dir=output/ \
  --dry-run      # validate only
  --from-step N  # partial re-run

Resume / Status / Copy

workflow resumeContinue after checkpoint
workflow statusShow pipeline progress (-v)
workflow listAvailable workflow YAML files
workflow copyCopy template to local dir
