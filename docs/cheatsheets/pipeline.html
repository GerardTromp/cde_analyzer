CDE Analyzer v0.6.0 — 3-Phase Pipeline

Pipeline Overview

Three-phase pipeline: raw NLM CDE JSON → cleaned, stripped outputs.

Phase 1: Instruments → Phase 2: Phrases → Phase 3: Branch
PhaseInputOutput
1allcde.jsoninst_stripped.json
2inst_stripped.jsoncurated_phrases.tsv
3allcde.json + patterns5 stripped JSON variants

Phase 1: Instrument Pipeline

instrument_pipeline.yaml

  1. mine_instruments — “as part of” patterns
  2. discover_abbreviations — abbreviated references
  3. discover_verbatim — verbatim + variant matching
  4. coalesce_patterns — remove subsumed
  5. validate_subsumption — empirical per-tinyId
  6. enrich_fields — add field distribution counts
  7. CHECKPOINT — curator reviews TSV
  8. strip_instruments — apply curated patterns
  9. sanity_check — diagnose remaining anchors

Key Outputs

inst_stripped.jsonInstruments removed from fields
curated.tsvCurator-approved patterns
inst_full.tsv / inst_sub.tsvStrip patterns (full / sub-group)

Phase 2: Phrase Pipeline

phrase_pipeline.yaml

  1. mine_phrases — k-mer mining (k=25→3)
  2. discover_verbatim — find verbatim occurrences
  3. coalesce_patterns — remove subsumed phrases
  4. field_analysis — add field counts + filtering
  5. CHECKPOINT — curator reviews TSV
  6. strip_phrases — apply curated patterns
  7. discovery_report — generate summary

Configuration

VariableDefaultPurpose
k_max25Longest phrase
k_min3Shortest phrase
min_field_count1Min CDEs per field
min_tokens2Min tokens

Key Outputs

curated_phrases.tsvCurator-approved phrases
dedup_phrases.tsvWhole-text duplicates (> k_max)
verbatim_phrases.tsvVerbatim surface forms

Curation Workflow

Human-in-the-loop iterative refinement at each checkpoint:

discover → edit → coalesce → validate
diagnose ← strip ← enrich ↑

Curation Tools

  • pattern_util --edit FILE — browser-based TSV editor
  • --harvest-to-supplementary — auto-ingest patterns
  • --promote-supplementary — local → global config
  • workflow resume — continue after checkpoint

Phase 3: Branching Strip

branching_strip.yaml — 13 steps, 6 outputs

  1. strip_MTSFPF — full instrument removal
  2. strip_MFSTPF — sub-group removal
  3. sub_on_full_MTSTPT — sub on full-stripped
  4. expand_temporal — generate temporal variants
  5. strip_temporal_* — temporal from 4 inputs
  6. strip_phrase_* — curated from 4 inputs
  7. quality_report — remnant detection

6 Output Branches

MTSFPFFull instrument removal only MFSTPFSub-group instrument removal only MFSFPTCurated phrases only (no instruments) MTSFPTFull instruments + phrases MFSTPTSub-group instruments + phrases MTSTPTAll instruments + phrases (maximum)

Split Temporal / Curated

Temporal case-insensitive → Curated case-sensitive

Temporal (~2,100 auto) stripped first with -I; then curated phrases with verbatim case matching.

Quick Execution

  -s input_json=allcde.json

# Phase 2
cde-analyzer workflow run \

  -s input_json=inst_stripped.json

cde-analyzer workflow run \
  branching_strip.yaml \
  -s input_json=allcde.json \
  -s inst_patterns_base=inst \
  -s phrase_patterns=curated.tsv

Workflow Engine, Configuration & Recipes

Workflow Engine

YAML-based pipeline orchestrator with checkpoints and resume.

YAML Structure

name: my_pipeline
variables:
  input_json: "data.json"
  output_dir: "output/"

steps:
  - name: mine_phrases
    action: phrase_miner
    args:
      input: "${input_json}"
      output: "${output_dir}"
      k-max: 25

  - name: curator_review
    action: checkpoint
    message: "Review patterns"

  - name: strip
    action: strip_phrases
    depends_on: [mine_phrases]

Key Features

variablesTemplate substitution via ${var}
depends_onExplicit step dependencies
checkpointPause for human curation
--set KEY=VALOverride variables at runtime
--from-stepStart from a specific step
--dry-runValidate without executing

Config Priority

Variables resolved in order (last wins):

  1. YAML variables: block (defaults)
  2. Auto-discovered {output_dir}/{name}_config.yaml
  3. CLI --set KEY=VALUE overrides

Auto-discovery: project-local config without modifying shared YAML.

Workflow Commands

Run a Workflow

cde-analyzer workflow run FILE.yaml \
  -s input_json=allcde.json \
  -s output_dir=phase1_output/ \
  -s workers=4

Resume After Checkpoint

cde-analyzer workflow resume \
  --state-file out/.workflow_state.json

Check Status

cde-analyzer workflow status \
  --state-file out/.workflow_state.json -v

List & Copy

# List available workflows
cde-analyzer workflow list
# Copy template to current directory
cde-analyzer workflow copy \
  instrument_pipeline -a my_pipe -d ./

Partial Re-run

cde-analyzer workflow run FILE.yaml \
  --from-step strip_phrases

Recipe: Discover & Curate

# 1. Discover patterns
cde-analyzer strip_discover \
  -i input.json -p inst.tsv \
  -e -o discovered.tsv

# 2. Curate (browser editor)
cde-analyzer pattern_util \
  --edit discovered.tsv

# 3. Coalesce (remove subsumed)
cde-analyzer pattern_util \
  -c discovered.tsv -o coal.tsv

# 4. Validate empirically
cde-analyzer pattern_util \
  -V coal.tsv -i input.json \
  -o validated.tsv

Recipe: Enrich & Strip

# 5. Enrich with field counts
cde-analyzer pattern_util \
  -A validated.tsv -i input.json \
  -o enriched.tsv

# 6. Strip matching text
cde-analyzer strip_phrases \
  -i input.json -p enriched.tsv \
  -o stripped.json

# 7. Check residuals
cde-analyzer diagnose_strip \
  -i stripped.json \
  -o residuals.tsv

Common One-Liners

# Subset by tinyId
cde-analyzer subset -i d.json \
  -l id1,id2 -o sub.json
# Count non-null definitions
cde-analyzer count -i d.json \
  --fields definitions.*.definition
# Extract for embeddings
cde-analyzer extract_embed \
  -i d.json -o embed.csv
# Merge pattern TSVs
cde-analyzer pattern_util \
  -M f1.tsv f2.tsv -o m.tsv
# Generate temporal variants
cde-analyzer pattern_util -T -o t.tsv
