Design Philosophy

Controlled synthetic CDEs isolate two measurable effects of instrument/boilerplate noise on embeddings:

  • Gravity — shared instruments pull unrelated CDEs together in embedding space
  • Drift — noise shifts a CDE away from its clean counterpart; terse CDEs drift more (noise is proportionally larger)

Predefined Strip Patterns

TSV Format

pattern  tinyIds  type  source  decision

Each pattern has tinyIds resolved from manifests for selective application.

TinyId Targeting

  • FSQA, HDQF, NCRF → tier 3 only (_t3 tinyIds)
  • Temporal phrases → tiers 1 + 3 (_t1, _t3)
  • Instrument names → tiers 2 + 3 (_t2, _t3)

Experiment Workflow

  1. Generate clean + noisy CDEs
  2. Embed all variants (multiple models)
  3. Cluster and measure gravity/drift
  4. Strip noise using predefined patterns
  5. Re-embed stripped CDEs
  6. Compare before/after clustering

Key Metrics

  • Drift distance: cosine(clean, noisy) per tier × verbosity
  • Gravity pull: cross-instrument cluster overlap
  • Strip recovery: % drift eliminated by stripping
  • Model sensitivity: drift variance across embedding models
