The 5 Decisions

Each row is a detected pattern. Decide what happens to it:

DecisionWhen to useEffect
keepReal instrument/scaleStripped (deleted)
removeFalse positiveExcluded entirely
modifyPartially correctModification text replaces pattern, then stripped
substituteReplace, don’t deleteModification text appears in output
followupNeeds more evaluationFlagged; counts as undecided

Modify vs Substitute

Modify changes what gets stripped (corrects the pattern text before deletion). Substitute changes what appears in the output (matched text is replaced, not deleted).

Blank decision = treated as keep (safe default).

Suggested Curator Workflow

  1. Sort by tinyId count (descending)
    Click tinyid_count header twice — highest-impact patterns first
  2. Evaluate top patterns → keep
    Select real instruments/phrases, press K
  3. Filter by group keyword
    For each “keep” pattern, filter the group_key column by a keyword from its tinyIds. This shows all related patterns — most will be extensions that remove semantic context → R
  4. Handle modify/substitute cases
    Press M or S, dbl-click modification, type correction
  5. Mass keep remaining
    Filter decision to blank, Ctrl+A, K
  6. Save
    Ctrl+S (server) or Save As (standalone)

Status bar shows tinyIds: decided/total to track coverage progress.

Groups, Advanced Features & Pipeline Integration

Merge Groups

When multiple patterns should share the same modification:

  1. Select 2+ related rows (Shift+click)
  2. Click [Group] — assigns G1 tag
  3. Set one member’s decision + modification
  4. Click [Propagate] — copies to all members

Before Propagation

# dec. modification pattern notes
47 modify in past 7 days I got enough sleep… G1
48 (empty) I had difficulty… G1
49 (empty) My sleep quality… G1

After Propagation

# dec. modification pattern notes
47 modify in past 7 days I got enough sleep… G1
48 modify in past 7 days I had difficulty… G1
49 modify in past 7 days My sleep quality… G1

Multiple groups (G1, G2, G3…) propagate independently. Propagation copies the source’s actual decision type — works with both modify and substitute.

Saving Your Work

Server Mode (recommended)

Launched with --edit FILE or cde_editor.pyz FILE:

  • Ctrl+S saves directly to original file
  • Unsaved changes trigger browser close warning

Standalone / Drag-Drop

Open editor without file, drag-drop a TSV:

  • Use Save As to download
  • Original file is not modified

Tips & Tricks

  • Undo is generous — 50 levels. Don’t fear mistakes.
  • Shift+click checkboxes for range selection — much faster.
  • Dbl-click a decision badge to edit via dropdown (single row).
  • The K shortcut is your friend — after triaging removes and modifies, filter to blank, Ctrl+A, K. Done.
  • TinyId columns collapse to 3 IDs + count badge. Click to expand.
  • Drag row numbers to reorder by drag-and-drop.
  • Column sorting — click any header.

After Curation

Return your curated TSV. The pipeline will:

  1. Merge your edits with auto-resolved patterns
  2. Apply decisions: keep → strip, modify → rewrite+strip, substitute → replace, remove → exclude
  3. Update ledger for future incremental runs
  4. Strip/substitute from CDE text fields

Multiple curators? Coordinator runs --merge-curation with inter-rater statistics.
