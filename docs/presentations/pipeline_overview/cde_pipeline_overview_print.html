2Project Intention

Processing CDE entries for embedding and clustering

Goal: Normalize CDE text fields so transformer embeddings capture clinical meaning, not shared boilerplate.

Problem: Raw CDE text contains repeated instrument names, temporal preambles, and instructional phrases that distort embedding space.

Impact:

  • CDEs sharing instruments cluster together regardless of clinical content
  • Shared boilerplate creates false semantic similarity
  • Terse CDEs experience proportionally larger semantic drift

Approach: Three-phase pipeline

  1. Detect repeated patterns automatically
  2. Curate with human-in-the-loop review
  3. Strip curated patterns, producing multiple variants

Scale: 22,743 CDEs processed; 553K characters removed (max variant)

5The Detection Challenge

  1. Variable surface forms: Same instrument — different punctuation, spacing, abbreviations
  2. Nested names: “NIH Toolbox” vs “NIH Toolbox General Life Satisfaction” — different granularity
  3. Temporal variants: “past 7 days” / “last seven days” / “the past week”
  4. Case sensitivity: Instrument names (Title Case) vs temporal patterns (case-insensitive)
  5. Context dependence: Pattern in an instrument name vs. in a definition sentence

Solution: Automated mining + human curation + multi-variant stripping

18TSV Editor: Motivation

Problem: Data corruption

  • Excel auto-converts tinyIds to dates/numbers
  • Excel strips leading zeros, changes encoding
  • Google Sheets reformats columns unpredictably

Problem: Restricted vocabulary

  • Decisions must be exactly: keep remove modify substitute
  • Free-text invites typos and inconsistency

Problem: Efficiency at scale

  • 1,480 patterns need batch operations
  • Keyboard shortcuts for rapid assignment
  • Filtering to focus on subsets

Solution

Browser-based TSV editor as Python zipapp (~59 KB, zero dependencies).

Works on any machine with Python 3.8+. No data leaves the local machine.

19TSV Editor Features

Decision assignment

  • Keyboard: K (keep), R (remove), M (modify), S (substitute)
  • Color-coded badges inline
  • Dropdown constraint on decision column

Filtering

  • Per-column text search
  • Numeric operators: =, !=, >=, <=
  • Decision dropdown filter
  • Clear all: Ctrl+Shift+F

Batch operations

  • Group propagation (selection-aware)
  • Shift+click range selection
  • Undo/redo: Ctrl+Z / Ctrl+Shift+Z

Visual cues

  • Field profile colors (def=blue, desig=orange, both=green)
  • Expandable tinyId pills per row
  • Group stripes (8 rotating hues)

Zipf pre-fill (v0.9.0)

--split-auto-remove pre-fills remove for low-priority patterns.

21Additional TSV Editing

Merge and Split are not used for curation but useful for pattern file management.

Merge patterns

--merge-patterns file1.tsv file2.tsv -o merged.tsv
  • Combine/deduplicate from different sources
  • Extracts columns by header name (not position)

Split operations

--split-priority needs_review.tsv --split-auto-remove
  • Zipf-based priority triage
  • Field-profile-based splitting

Pattern utility commands

  • --coalesce-variants: subsumption + prefix trie
  • --field-analysis: add field counts
  • --validate-subsumption: empirical check
  • --expand-temporal-seeds: seed → variants
  • --harvest-to-supplementary: ingest residuals

23Key Takeaways

  1. Three-phase pipeline with human-in-the-loop curation at each phase
  2. Descending k-mer mining with masking prevents redundant detection
  3. Curation ledger enables incremental improvement — effort compounds across runs
  4. Zipf triage separates domain-specific patterns from common English
  5. Five stripped variants for downstream comparison (MT+ST removed as equivalent)
  6. min_parent_tinyids is the single most influential parameter (18.6x)
  7. Standalone editor prevents Excel data corruption during curation
