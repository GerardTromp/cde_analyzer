Why Synthetic Data?

Problem: Real CDE text contains instrument names, temporal phrases, and boilerplate that distort embedding-based clustering.
Solution: Controlled synthetic CDEs with known noise injections — measure the effect, then measure the cure.

Two Measurable Effects

  • Gravity Shared instruments pull unrelated CDEs together in embedding space
  • Drift Noise shifts a CDE away from its clean counterpart; terse definitions drift most

TierSuffixInjection
T1 Light_t1Temporal phrase only

Injection Points

  • Name (designation 1): prepend sub-scale
  • Question (designation 2): prepend temporal
  • Definition: append temporal clause + anchor phrase

Example: synNR01T (Terse)

Clean name: Stroke Onset Time

T2 name: NESS Cerebrovascular Module - Stroke Onset Time

T3 def: ...interval from symptom onset, assessed during the past 6 months. A component of the NESS. Based on the NCRF.

After stripping, CDEs should revert to clustering by semantic content rather than by shared instrument.

TSV Format

pattern  tinyIds  type  source  decision

TinyId-Based Targeting

  • Each pattern lists which CDE tinyIds it applies to
  • Resolved from dataset manifests at generation time
  • Enables selective stripping per CDE
Tier-aware: FSQA/HDQF/NCRF targets _t3 only. Temporal targets _t1 + _t3. Instruments target _t2 + _t3.

Experiment Workflow

1. Generate → 2. Embed → 3. Cluster → 4. Strip → 5. Re-embed → 6. Compare

Key Metrics

  • Drift distance — cosine(clean, noisy) per tier × verbosity
  • Gravity pull — cross-instrument cluster overlap
  • Strip recovery — % of drift eliminated by stripping
  • Model sensitivity — drift variance across embedding models

Expected Results

  • Terse CDEs: largest drift, most sensitive to noise
  • Expansive CDEs: minimal drift, noise absorbed
  • Gravity strongest when instrument text dominates definition
  • Strip recovery should approach 100% for known patterns

Cross-Model Evaluation

Research Questions

  1. Which embedding models are most sensitive to instrument noise?
  2. Does stripping improve clustering quality uniformly across models?
  3. Is the verbosity × drift interaction model-dependent?
  4. Do gravity effects persist in high-dimensional models?

Generation Commands

# Generate neurology base (60 clean CDEs)
python scripts/generate_health_neuro.py \
  -o data/synthetic_qc/health_neuro/health_neuro.json --pretty

# Generate neurology drift (180 noisy CDEs)
python scripts/generate_health_neuro_drift.py \
  -s data/synthetic_qc/health_neuro/health_neuro.json \
  -o data/synthetic_qc/health_neuro_drift/health_neuro_drift.json --pretty

# Generate strip patterns for ALL datasets (63 patterns)
python scripts/generate_synthetic_strip_patterns.py \
  -o data/synthetic_qc/strip_patterns/

# Run branching strip with synthetic patterns
cde-analyzer strip_branching \
  -i combined_noisy.json -d output/ \
  --inst-full-patterns data/synthetic_qc/strip_patterns/inst_full_patterns.tsv \
  --inst-sub-patterns data/synthetic_qc/strip_patterns/inst_sub_patterns.tsv \
  --phrase-patterns data/synthetic_qc/strip_patterns/phrase_patterns.tsv

Summary

What We Built

  • 840 CDEs across 8 controlled datasets
  • 12 instrument families with 29 sub-scales
  • 3 noise tiers (light / medium / heavy)
  • 3 verbosity tiers (terse / informational / expansive)
  • 63 strip patterns for complete noise removal
  • 10 generator scripts for reproducibility

What It Enables

  • Quantify gravity and drift effects per model
  • Validate strip pipeline effectiveness
  • Compare embedding model robustness
  • Isolate verbosity vs. domain confounds
  • Benchmark clustering algorithms
  • Regression testing for pipeline changes
